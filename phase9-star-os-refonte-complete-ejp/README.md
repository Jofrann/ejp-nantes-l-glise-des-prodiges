# PHASE 9 — STAR OS EJP Nantes
## Dossier maître de refonte complète de l’espace serviteur

Ce dossier remplace les anciennes approches centrées sur les départements.

La plateforme interne EJP doit devenir un **poste de travail personnel** pour chaque serviteur, appelé **STAR OS**.

**STAR = Serviteur Travaillant Activement pour le Royaume.**

La logique centrale n’est plus :

> Je choisis un département → je cherche quoi faire.

La logique centrale devient :

> Je me connecte → j’entre dans mon espace personnel → je vois mon rythme, mes présences, mes formations, ma croissance, mes rendez-vous, mes ressources, mes responsabilités actives.

Les départements restent utiles, mais ils ne sont plus l’entrée principale.  
Ils deviennent des sources d’attribution, de responsabilité, de données et de supervision.

## Règle de vérité

Si un serviteur se connecte et ne comprend pas en moins de 5 secondes :
- ce qu’il doit faire aujourd’hui ;
- comment confirmer sa présence ;
- où continuer sa formation ;
- où demander de l’aide ;
- où trouver ses responsabilités ;

alors l’application n’est pas encore assez claire.
